# Proyecto

# Ingever Asociados S.A.S.

## Descripción General

El presente proyecto consiste en el desarrollo del nuevo sitio web corporativo para **Ingever Asociados S.A.S.**, una empresa especializada en soluciones de ingeniería, fabricación, montaje y mantenimiento de estructuras metálicas, puentes grúa y equipos industriales.

El objetivo no es únicamente reemplazar el sitio web existente, sino construir una plataforma moderna, rápida, escalable y preparada para crecer durante los próximos años.

Este proyecto está siendo desarrollado utilizando tecnologías modernas del ecosistema JavaScript, priorizando la mantenibilidad, la experiencia de usuario y la facilidad para futuras ampliaciones.

---

# Objetivos del Proyecto

Los objetivos principales son:

- Modernizar la imagen digital de la empresa.
- Presentar claramente los servicios ofrecidos.
- Facilitar la comunicación con clientes potenciales.
- Permitir el envío de solicitudes de cotización.
- Incorporar un portal de empleo para recibir hojas de vida.
- Mejorar la experiencia de navegación en dispositivos móviles.
- Optimizar el rendimiento del sitio.
- Preparar la plataforma para futuras funcionalidades.

---

# Alcance

El proyecto contempla el desarrollo del frontend completo del sitio web.

Incluye:

- Landing Page corporativa.
- Sección institucional.
- Presentación de servicios.
- Proyectos realizados.
- Clientes.
- Información de contacto.
- Formulario de cotización.
- Portal de empleo.
- Formularios responsivos.
- Navegación entre páginas.
- Optimización para SEO.
- Integración con Git y Netlify.

No incluye, por el momento:

- Panel administrativo.
- Base de datos.
- Gestión de usuarios.
- Sistema de autenticación.
- Backend propio.
- Dashboard administrativo.

Estas funcionalidades podrán añadirse en futuras fases del proyecto.

# Metodología

El proyecto sigue una metodología incremental.

Cada funcionalidad debe cumplir el siguiente flujo:

1. Planificación.
2. Diseño.
3. Desarrollo.
4. Pruebas.
5. Documentación.
6. Commit.
7. Push.
8. Deploy automático.

No se deben desarrollar múltiples funcionalidades críticas simultáneamente.

Cada módulo debe quedar completamente funcional antes de iniciar el siguiente.

# Principios de Desarrollo

Durante todo el proyecto deberán respetarse los siguientes principios:

- Simplicidad antes que complejidad.
- Reutilizar antes que duplicar.
- Modularizar antes que agrupar.
- Documentar antes de finalizar.
- Optimizar únicamente cuando sea necesario.
- Mantener una arquitectura consistente.
- Priorizar la experiencia del usuario.
